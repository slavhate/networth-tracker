# Print-Friendly HTML Report Export — Design Spec

**Date:** 2026-04-10
**Context:** Personal/hobby project — add a self-contained HTML report export that works offline and prints cleanly.

---

## 1. Overview

Add a "Print Report" button to the sidebar that generates and downloads a self-contained HTML file (`networth-report-YYYY-MM-DD.html`) containing the user's complete financial snapshot. No backend changes. No new dependencies.

---

## 2. Architecture

### Files Changed

| File | Change |
|------|--------|
| `frontend/src/utils/reportGenerator.js` | New file — all report generation logic |
| `frontend/src/components/Layout.jsx` | Add "Print Report" button to sidebar |

### Flow

1. User clicks "Print Report" button in sidebar (next to Backup/Restore)
2. Button text changes to "Generating…" and becomes disabled
3. `generateReport(isPrivate)` is called from `reportGenerator.js`, where `isPrivate` is read from `PrivacyContext` in `Layout.jsx` and passed in as a boolean
4. Fetches data from 8 API endpoints in parallel:
   - `GET /api/dashboard`
   - `GET /api/assets`
   - `GET /api/liabilities`
   - `GET /api/bank-accounts`
   - `GET /api/mutual-funds`
   - `GET /api/equities`
   - `GET /api/insurances`
   - `GET /api/goal`
5. Assembles a self-contained HTML string with inline CSS
6. Creates a Blob and triggers browser download as `networth-report-YYYY-MM-DD.html`
7. Button returns to normal state
8. On any fetch failure: show error message (same pattern as existing pages), abort, restore button

---

## 3. Report Content

Sections appear in this order:

1. **Header** — "Net Worth Financial Report · [full date e.g. 10 April 2026]"
2. **Summary strip** — 4 stat cards: Net Worth, Total Assets, Total Liabilities, Debt-to-Asset Ratio
3. **Net Worth Goal** — goal name, target amount (in crores), progress bar, current % complete. Omitted entirely if no goal is set.
4. **Assets** — table: Name | Category | Value — bold subtotal row at bottom
5. **Liabilities** — table: Name | Category | Amount | Interest Rate — bold subtotal row
6. **Bank Accounts** — table: Bank | Account Type | Account No (last 4 digits) | Balance — bold subtotal row
7. **Mutual Funds** — table: Fund Name | Units | Avg NAV | Current NAV | Current Value — bold subtotal row
8. **Equities** — table: Symbol | Market | Quantity | Avg Buy Price | Current Price | Current Value — bold subtotal row
9. **Insurances** — table: Policy Name | Type | Insurer | Premium | Frequency — no monetary subtotal (premiums are periodic, not a balance)
10. **Footer** — "Generated by Net Worth Tracker on [ISO timestamp]"

**Empty sections:** If a section has no entries, it still renders with its header and a single "No entries" row. Section is never omitted.

**Privacy mode:** A single `isPrivate` boolean is passed into `generateReport()`. When `true`, every formatted currency value in the report is replaced with `₹ ••••••`. Applied as a single formatting switch, not scattered conditional logic.

---

## 4. Styling

All CSS is inlined in the HTML `<style>` tag — no external stylesheets, fonts, or scripts.

- **Page**: White background, black text, max-width 900px centered
- **Summary cards**: 4 cards in a row (wraps to 2×2 on narrow viewports)
- **Section headers**: Dark background band (#1e293b), white text, uppercase label
- **Tables**: Full-width, alternating row shading, left-aligned text, right-aligned numbers, bold subtotal rows
- **Number formatting**: `Intl.NumberFormat('en-IN', { style: 'currency', currency: 'INR', minimumFractionDigits: 0, maximumFractionDigits: 0 })` — matches the app's Indian locale formatting
- **`@media print`**: Hides browser chrome, forces black-on-white, inserts `page-break-before: always` before each section (Assets onwards) so each section starts on a fresh page

---

## 5. What Is NOT Changing

- No backend changes
- No new npm dependencies
- No new API endpoints
- No changes to existing pages or data flow
- Charts are not included in the report (tables only — charts don't render usefully in static HTML)
